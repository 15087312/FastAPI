# 🚀 端口占用自动切换功能说明

## ✨ 功能特性

### 1. **端口占用检测**
- 启动时自动检测配置端口是否被占用
- 如果被占用，自动从 8001、8002... 依次尝试找到可用端口
- 支持自定义最大尝试次数（默认 10 次）

### 2. **动态端口返回**
- 新增 `/config` 端点，返回实际运行的端口信息
- 前端可通过该接口动态获取服务地址
- API 文档中的服务器地址也会动态更新

### 3. **日志提示**
- 端口被占用时会显示警告日志
- 找到新端口后会显示成功日志
- 无法找到可用端口时会抛出错误

## 🛠️ 配置说明

### **环境变量（.env 文件）**

```env
# 服务器监听地址和端口
HOST=0.0.0.0
PORT=8000
DEBUG=True
```

- `HOST`: 监听地址，默认 `0.0.0.0`（允许外部访问）
- `PORT`: 期望的端口，默认 `8000`
- `DEBUG`: 调试模式，默认 `True`

### **端口查找逻辑**

```python
# 伪代码示例
期望端口 = 8000
如果 8000 被占用：
    尝试 8001
    如果 8001 被占用：
        尝试 8002
        ...
        最多尝试 10 次（到 8009）
```

## ⚠️ 注意事项

1. **生产环境配置**
   - 生产环境应使用固定的端口配置
   - 通过环境变量或配置文件管理端口
   - 不建议在生产环境使用自动端口切换

2. **防火墙设置**
   - 确保选择的端口在防火墙允许范围内
   - Windows 可能需要允许 Python 通过网络通信

3. **Docker 端口映射**
   - 如果使用 Docker，需要在 docker-compose.yml 中映射所有可能的端口
   - 或者只映射期望的端口，让应用在容器内切换

4. **负载均衡器**
   - 如果前面有 Nginx 等负载均衡器，需要配置反向代理规则
   - 确保所有可能的端口都能被访问
