## 🚀 快速开始

### 1. 复制核心文件

将以下文件复制到你的项目：

```
app/core/
├── config_generic.py          # 通用配置类
├── model_factory.py           # 模型工厂
└── model_configs.json         # 模型配置示例（可选）

app/services/
└── generic_inventory_service.py  # 通用库存服务（可选）
```

### 2. 修改环境变量

在 `.env` 文件中添加：

```bash
# 通用模型配置
ENABLED_MODELS=Product,ProductStock,Warehouse,Order
USE_BUILTIN_INVENTORY_MODELS=True
USE_CUSTOM_MODELS=False
CUSTOM_MODEL_CONFIG_PATH=app/core/model_configs.json

# 业务逻辑配置
ENABLE_REDIS_CACHE=true
ENABLE_BLOOM_FILTER=true
ENABLE_IDEMPOTENCY=true
```

### 3. 使用动态模型

```python
from app.core.model_factory import create_product_model, create_product_stock_model

# 创建模型类
Product = create_product_model()
ProductStock = create_product_stock_model()

# 使用模型
print(f"Product 表名：{Product.__tablename__}")
print(f"ProductStock 表名：{ProductStock.__tablename__}")
```

## ⚙️ 模型配置

### 方式一：使用内置配置

适用于标准电商库存场景：

```bash
# .env
USE_BUILTIN_INVENTORY_MODELS=True
ENABLED_MODELS=Product,ProductStock,InventoryReservation,InventoryLog
```

支持的内置模型：
- `Product` - 商品模型
- `ProductStock` - 商品库存
- `InventoryReservation` - 库存预占
- `InventoryLog` - 库存日志
- `IdempotencyKey` - 幂等性键

### 方式二：使用 JSON 配置文件

适用于需要自定义字段和表的场景：

```bash
# .env
USE_CUSTOM_MODELS=True
CUSTOM_MODEL_CONFIG_PATH=config/my_models.json
```

JSON 配置文件格式：

```json
{
  "Product": {
    "table_name": "products",
    "fields": {
      "id": {
        "type": "BigInteger",
        "primary_key": true,
        "autoincrement": true
      },
      "name": {
        "type": "String",
        "max_length": 255,
        "nullable": false,
        "comment": "商品名称"
      }
    }
  }
}
```

### 方式三：混合模式

同时使用内置和自定义模型：

```bash
# .env
USE_BUILTIN_INVENTORY_MODELS=True
USE_CUSTOM_MODELS=True
ENABLED_MODELS=Product,ProductStock,CustomModel1,CustomModel2
CUSTOM_MODEL_CONFIG_PATH=config/custom_models.json
```

## 🔧 业务逻辑配置

### 启用/禁用功能

```bash
# Redis 缓存
ENABLE_REDIS_CACHE=true

# 布隆过滤器（用于快速判断商品是否存在）
ENABLE_BLOOM_FILTER=true

# 幂等性检查（防止重复请求）
ENABLE_IDEMPOTENCY=true

# Kafka 异步同步数据库
ENABLE_KAFKA_SYNC=false
```

### 性能调优

```bash
# Redis 缓存过期时间（秒），0 表示永不过期
REDIS_CACHE_TTL=0

# 批量操作最大数量
BATCH_OPERATION_MAX_SIZE=100

# Lua 脚本预注册（提升性能）
PRE_REGISTER_LUA_SCRIPTS=true
```

## 🔄 迁移到现有项目

### 步骤 1：准备环境

```bash
# 安装依赖
pip install fastapi sqlalchemy redis pydantic python-dotenv
```

### 步骤 2：复制核心代码

```bash
# 复制配置和模型工厂
cp -r app/core /your/project/app/core

# 复制服务层（可选）
cp app/services/generic_inventory_service.py /your/project/app/services/

# 复制基础设施
cp -r app/db /your/project/app/db
cp app/core/redis.py /your/project/app/core/
```

### 步骤 3：配置环境变量

```bash
# 复制并修改 .env
cp .env.example /your/project/.env
vi /your/project/.env

# 修改数据库连接信息
POSTGRES_USER=your_user
POSTGRES_PASSWORD=your_password
POSTGRES_HOST=your_host
POSTGRES_DB=your_db

# 配置模型
ENABLED_MODELS=YourModel1,YourModel2
USE_CUSTOM_MODELS=True
CUSTOM_MODEL_CONFIG_PATH=config/your_models.json
```

### 步骤 4：创建模型

```python
# main.py 或 models/__init__.py
from app.core.model_factory import ModelFactory

# 创建所有启用的模型
models = ModelFactory.create_all_models()

# 或使用单个创建
YourModel = ModelFactory.create_model("YourModel")
```

### 步骤 5：运行数据库迁移

```bash
# 使用 Alembic 创建表
alembic revision --autogenerate -m "Create custom tables"
alembic upgrade head
```

## 🔍 常见问题

### Q1: 如何添加新字段？

编辑 JSON 配置文件，在对应模型的 `fields` 中添加：

```json
{
  "Product": {
    "fields": {
      "new_field": {
        "type": "String",
        "max_length": 100,
        "nullable": true,
        "comment": "新字段说明"
      }
    }
  }
}
```

然后运行：
```bash
alembic revision --autogenerate -m "Add new_field to products"
alembic upgrade head
```

### Q2: 如何修改字段类型？

直接修改 JSON 配置中的 `type` 字段，然后运行数据库迁移。

⚠️ 注意：某些类型转换可能导致数据丢失，请谨慎操作。

### Q3: 如何在代码中使用动态模型？

```python
from app.core.model_factory import create_product_model

Product = create_product_model()

# 查询
from sqlalchemy import select
stmt = select(Product).where(Product.id == 1)
product = db.execute(stmt).scalar_one_or_none()

# 创建
new_product = Product(name="商品名称", sku="SKU001")
db.add(new_product)
db.commit()
```
